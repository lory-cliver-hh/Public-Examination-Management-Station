# 公考管理系统需求文档

## 1. 文档信息

- 项目名称：公考管理系统
- 文档类型：需求文档 / 产品方案初稿
- 适用阶段：产品定义、原型设计、技术实施前期
- 当前定位：单用户、本地优先、网页端学习管理系统

## 2. 项目背景

当前公考复习过程通常分散在多个工具中：

- 课程视频在夸克网盘
- 学习记录零散写在纸质笔记、微信、备忘录或 Excel 中
- 每日学习计划、复盘、进度统计缺少统一视图

这会导致几个明显问题：

- 不清楚今天实际学了什么
- 不清楚哪一科投入时间最多、哪一科推进最慢
- 打开课程和记录学习是割裂的两个动作
- 复习资料、视频课程、学习日志之间无法形成闭环

因此需要一个网页端系统，将“学习内容管理”和“学习过程记录”统一起来，并通过课程分享链接与夸克网盘连接。

## 3. 设计目的

本系统的设计目的不是替代夸克网盘，而是在夸克之上建立一层学习管理界面，实现以下目标：

1. 统一记录每天学习了什么内容、学了多久、学到哪里。
2. 将公考课程按科目、模块、章节、课时进行结构化管理。
3. 为每个课时绑定一个夸克分享链接，实现一键跳转观看。
4. 形成学习历史、学习统计、阶段复盘和后续计划。
5. 让复习过程从“零散执行”变成“可视、可追踪、可复盘”的系统化流程。

## 4. 目标用户

- 主要用户：你自己
- 使用场景：个人备考、公考课程跟学、日常复盘、阶段冲刺
- 使用设备：以桌面浏览器为主，兼顾手机浏览
- 使用强度：高频、长期、单用户、私有数据

## 5. 产品定位

这是一个“个人学习驾驶舱”而不是“泛化在线教育平台”。

它应该具备以下定位特征：

- 单用户优先，不引入复杂权限系统
- 本地优先，先解决稳定可用，再考虑云同步
- 网页端操作轻量，重点是高频记录与快速打开课程
- 以学习管理为核心，视频播放继续交给夸克网页端

## 6. 核心需求概述

### 6.1 必须解决的问题

- 我今天学了什么
- 我某个章节的视频在哪里
- 我这周/这月总共学了多久
- 哪些课程已经完成，哪些还没开始
- 明天该继续学哪一节

### 6.2 产品必须具备的能力

- 课程结构管理
- 课时级夸克链接管理
- 学习记录录入
- 学习进度统计
- 日报/周报式复盘
- 考试倒计时提醒

## 7. 业务边界与约束

### 7.1 系统边界

系统负责：

- 存储课程结构
- 存储课时链接
- 记录学习行为
- 展示统计数据
- 提供计划和复盘界面

系统不负责：

- 直接托管视频文件
- 直接嵌入夸克播放器控制播放
- 逆向获取夸克私有文件目录
- 依赖夸克内部未公开接口

### 7.2 夸克集成约束

采用“分享链接跳转”的集成方式：

- 每个课时保存一个 `share_url`
- 用户点击“去观看”后，新标签页打开夸克网页端
- 如果链接失效，需要在系统中更新对应课时链接
- 如果链接需要提取码，可额外保存 `share_code`

该方案的优点是简单、稳定、维护成本低，适合个人长期使用。

## 8. 典型使用流程

### 8.1 日常学习流程

1. 打开系统首页，查看“今日计划”和“待继续课程”
2. 点击某个课时的“打开夸克继续学习”
3. 跳转到夸克网页端观看视频
4. 学完后回到系统，记录学习时长、完成度和笔记
5. 系统自动累计当日学习数据并更新课程进度

### 8.2 复盘流程

1. 在日历或时间轴中查看近 7 天学习记录
2. 统计各科目学习时长和完成进度
3. 补充今日总结与明日计划
4. 调整接下来优先学习的课程模块

## 9. 功能需求

### 9.1 首页仪表盘

首页用于快速进入当天学习状态，建议包含以下模块：

- 今日学习总览
- 今日已学时长
- 今日完成课时数
- 待继续课程
- 最近学习记录
- 本周学习趋势
- 考试倒计时卡片
- 快速新增学习记录

### 9.1.1 考试倒计时组件

该组件用于强化备考节奏感，建议固定显示在首页右下角、右侧栏顶部或悬浮区域。

建议支持以下能力：

- 自定义考试名称，如国考、省考、事业编、选调生等
- 自定义考试日期与时间
- 自动显示距离考试还有多少天 / 小时
- 支持多个考试倒计时并设置优先级
- 支持标记“当前主目标考试”

设计要求：

- 视觉上应明显，但不能压过首页主要学习内容
- 默认展示最近的一场考试
- 可折叠，可展开查看全部考试节点
- 移动端应收纳为顶部提醒条或小型卡片

### 9.2 课程管理

课程数据建议按以下层级组织：

- 科目：行测 / 申论 / 常识 / 判断推理 / 资料分析等
- 模块：如判断推理下的图形推理、逻辑判断等
- 章节：具体课程章节
- 课时：可直接跳转的视频单元

每个课时建议字段：

- 课时标题
- 所属科目
- 所属模块
- 所属章节
- 夸克分享链接
- 提取码（可选）
- 视频时长（可选）
- 当前状态：未开始 / 学习中 / 已完成
- 个人备注

### 9.3 学习记录

学习记录是系统的核心数据，建议至少包含：

- 学习日期
- 开始时间
- 结束时间
- 学习时长
- 对应课时
- 学习进度
- 学习笔记
- 当天反思

### 9.4 学习统计

统计页建议至少提供：

- 每日学习时长趋势
- 各科目学习时长占比
- 课程完成率
- 最近 7 天 / 30 天学习情况
- 已完成、进行中、未开始课时数量

### 9.5 计划与复盘

用于形成闭环管理：

- 今日计划
- 明日计划
- 每周目标
- 本周复盘
- 阶段重点模块标记

### 9.6 公考信息更新（预留模块）

该模块用于后续接入考试资讯与通知汇总，但当前阶段只做需求预留，不进入 MVP。

未来可扩展方向：

- 汇总国考、省考、事业编、选调生等招考通知
- 展示报名时间、笔试时间、资格审查、面试公告等关键节点
- 按地区筛选信息，如全国、各省、市、县
- 标记“最新发布”“即将截止”“重点关注”

潜在实现方式：

- 后续可接入定向爬虫或信息抓取服务
- 也可先采用人工录入或订阅源聚合

当前阶段约束：

- 暂不做爬虫开发
- 暂不接入自动资讯采集
- 暂不承诺信息实时性

## 10. 页面结构设计

建议采用以下页面结构：

### 10.1 页面清单

- 首页 / 仪表盘
- 课程中心
- 课时详情页
- 学习记录页
- 学习统计页
- 日历复盘页
- 公考信息更新页（预留）
- 设置页

### 10.2 导航结构

- 左侧主导航：首页、课程、记录、统计、复盘、设置
- 顶部快捷区：搜索、快速记录、今日学习时长、最近学习入口

### 10.3 页面关系

- 首页负责概览和快速进入
- 课程页负责找课、跳转、更新状态
- 记录页负责补录和查看学习流水
- 统计页负责看趋势与科目分布
- 复盘页负责日历、总结和阶段计划

## 11. 视觉与交互设计方向

本项目不应做成“通用后台模板”的样子，而应更像“个人学习驾驶舱”。

### 11.1 视觉目标

- 专注
- 清晰
- 有学习氛围
- 信息层级强
- 高密度但不拥挤

### 11.2 建议视觉风格

- 主色方向：米白底或浅灰底，搭配深墨绿 / 深蓝作为主色
- 强调色：暖橙或赤陶色，用于“继续学习”“今日重点”“完成状态”
- 风格关键词：纸质笔记感、课程目录感、学习面板感
- 避免风格：大面积紫色、纯模板式卡片墙、过度渐变、假大空营销风

### 11.3 页面呈现原则

结合已安装的 `web-design-patterns` 技能，页面应遵守以下原则：

- 必须有一个视觉主导区域，不要所有模块同等权重
- 卡片不要全部一模一样，应按重要程度区分大小和层级
- CTA 文案必须具体，如“打开夸克继续学习”“记录今日学习”“完成本章”
- 不做对称堆砌式布局，首页应有明确的阅读顺序
- 图表、卡片、列表之间要有节奏，不要做成流水账式面板

### 11.4 首页预期效果

首页建议采用“左主右辅”的结构：

- 左侧主区域：今日计划、待继续课程、最近学习记录
- 右侧辅助区域：本周统计、学习时长、完成率、小型趋势图、考试倒计时

其中“今日重点课程”应作为最大信息块突出显示，形成首页第一视觉落点。

“考试倒计时”不应做成喧宾夺主的大横幅，更适合做成有存在感的小型高优先级组件，用于持续提醒备考节点。

### 11.5 课程页预期效果

- 左侧为课程树
- 右侧为章节/课时列表
- 课时列表可直接显示状态、时长、备注和“打开夸克”按钮
- 当前学习中的课时应有明显强调样式

### 11.6 移动端预期效果

- 保留核心功能：查看计划、打开课程、记录学习
- 将复杂统计折叠为简版卡片
- 课程树改为折叠面板或抽屉结构

## 12. 信息架构与数据结构建议

### 12.1 主要实体

- Subject：科目
- Module：模块
- Chapter：章节
- Lesson：课时
- LessonLink：课时链接
- StudyRecord：学习记录
- DailyReview：日复盘
- WeeklyGoal：周目标
- ExamCountdown：考试倒计时
- InfoNotice：资讯通知（预留）

### 12.2 关键数据关系

- 一个科目包含多个模块
- 一个模块包含多个章节
- 一个章节包含多个课时
- 一个课时对应一个或多个资源链接
- 一条学习记录关联一个课时
- 一个考试倒计时独立存在，可在首页高优先级展示
- 一条资讯通知可关联考试类别、地区和发布日期

## 13. 技术架构建议

### 13.1 总体架构

建议采用单体全栈架构：

- 前端：网页页面、交互逻辑、图表展示
- 后端：课程数据、学习记录、统计接口
- 数据库：本地存储课程结构和学习数据

### 13.2 推荐技术选型

- 前端框架：Next.js
- 开发语言：TypeScript
- UI 样式：Tailwind CSS
- 组件基础：shadcn/ui 或自定义轻量组件
- 数据层：Prisma
- 数据库：SQLite（前期） / PostgreSQL（后期可升级）
- 图表：ECharts 或 Recharts
- 表单校验：Zod + React Hook Form

### 13.3 选型原因

- Next.js 适合快速搭建网页端与接口一体化项目
- TypeScript 便于长期维护
- SQLite 足够支撑单用户场景，部署简单
- Prisma 方便后续从 SQLite 升级到 PostgreSQL
- 图表库可快速实现学习趋势和占比统计

## 15. 非功能需求

### 15.1 性能

- 首页应快速打开
- 课程列表和学习记录查询需流畅
- 常用页面优先保证桌面端体验

### 15.2 可维护性

- 课程结构与记录模块应解耦
- 链接管理与学习记录逻辑分离
- 文档、数据库结构、页面结构需保持一致

### 15.3 可扩展性

后续应支持：

- Excel / CSV 批量导入课程
- 课程标签和筛选
- 番茄钟或专注计时
- 错题本或笔记模块
- 云同步或多设备访问
- 倒计时提醒增强
- 公考资讯自动采集

## 16. MVP 范围

第一阶段建议只做最核心闭环：

1. 课程结构管理
2. 课时夸克链接绑定
3. 学习记录录入
4. 首页仪表盘
5. 学习统计基础版
6. 基础考试倒计时

当前明确不纳入 MVP 的内容：

- 自动爬取各地公考/事业编通知
- 资讯聚合页的自动更新能力
- 爬虫调度、去重、异常处理等采集系统

只要这六项跑通，系统就已经具备实际使用价值。

## 18. 最终预期呈现效果

最终呈现的系统应当具备以下感受：

- 打开首页后，立刻知道今天该学什么
- 一眼看到距离目标考试还有多久
- 点击课时后，可以直接去夸克继续学习
- 学完后，能迅速记录内容和时长
- 一周后回看，能看到清晰的学习轨迹
- 整个界面像“个人复习面板”，而不是冷冰冰的通用后台

从视觉上，它应该更像“学习驾驶舱 + 纸质笔记的数字化界面”，强调专注、节奏和计划感，而不是做成一个普通课程表网站。

## 19. 结论

该项目具备明确的使用价值、清晰的功能闭环和可控的技术复杂度，适合按“需求文档 -> 线框图 -> 原型 -> MVP 开发”的顺序推进。

当前最合适的下一步是：

1. 基于本需求文档输出页面原型结构
2. 明确数据库表结构
3. 初始化项目骨架并开始开发 MVP
